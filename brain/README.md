# Freelance Market Validation

Objetivo del proyecto: validar en una semana si tiene sentido buscar clientes como autónomo en Alemania con una oferta basada en Angular, design systems, internal tools, automatización, APIs y AI-assisted development.

## Cómo usar este repositorio

1. Lee primero `00-resumen-ejecutivo.md`.
2. Monta el portfolio mínimo con `01-portfolio/portfolio-minimo-48h.md`.
3. Define tus servicios con `02-servicios/servicios-principales.md` y `02-servicios/paquetes-comerciales.md`.
4. Usa `04-leads/ofertas-reales.md` y `04-leads/perfiles-para-cold-calling.md` para contactar leads esta semana.
5. Copia las plantillas de `05-outreach/`.
6. Revisa `06-finanzas/` antes de tomar decisiones grandes.
7. Ejecuta el plan de `07-plan-semanal/`.
8. Decide con `09-decision/`.

## Hipótesis principal

No venderte primero como “Angular developer genérico”. El mercado te compararía con muchos perfiles similares. Venderte como consultor técnico externo para **internal tools, frontend systems y automatización operativa** tiene mejor narrativa comercial porque conecta con ahorro, velocidad y reducción de trabajo manual.

## Nichos principales

| Nicho | Por qué tiene sentido | Oferta inicial |
|---|---|---|
| Internal tools | Muchas pymes tienen procesos entre Excel, email y herramientas sueltas | MVP interno o automation sprint |
| Angular / design systems | Encaja con tu experiencia y genera contratos más largos | Stabilization sprint o design system rollout |
| AI-assisted development | Empresas usan Copilot/IA sin método claro | Adoption kit, workshop y guardrails |
| Workflow automation | Dolor claro y ROI más fácil | n8n/Make/Power Automate sprint |
| API integrations | Muy común en operaciones, ventas, soporte y finanzas | Integration workflow + documentación |
