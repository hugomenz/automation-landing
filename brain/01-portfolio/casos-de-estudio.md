# Casos de estudio

## Caso A: Modernización Angular y sistema de componentes

### Situación

El equipo tiene componentes duplicados, inconsistencias visuales y ciclos lentos de entrega.

### Intervención

- Auditoría del frontend.
- Definición de componentes base.
- Storybook/documentación.
- Reglas de uso.
- Pruebas básicas.

### Resultado demostrable

- Menos duplicación.
- UI más consistente.
- Mejor onboarding para developers.
- Base clara para nuevas features.

### Cómo contarlo

> Convertí UI repetida en un sistema reusable, documentado y preparado para crecer sin crear deuda en cada sprint.

## Caso B: Automatización de soporte con n8n y APIs

### Situación

El equipo de soporte clasifica tickets manualmente y mueve información entre herramientas.

### Intervención

- Workflow con n8n.
- Clasificación automática.
- Resumen con IA.
- Routing a Slack/helpdesk.
- Logging y control humano.

### Resultado demostrable

- Menos tareas repetitivas.
- Mejor trazabilidad.
- Respuestas más rápidas.
- Menor riesgo de errores manuales.

### Cómo contarlo

> Diseñé una automatización útil para soporte, con humano en el loop y control de errores, no un bot sin contexto.

## Caso C: GitHub Copilot Adoption Kit

### Situación

La empresa tiene Copilot o quiere usar IA, pero cada developer lo usa distinto y no hay estándares.

### Intervención

- Playbook de uso.
- Ejemplos por tarea.
- PR checklist.
- Reglas de seguridad.
- Workshop de adopción.

### Resultado demostrable

- Uso más consistente.
- Menos outputs sin revisar.
- Mejor onboarding.
- Menos fricción con IA en el equipo.

### Cómo contarlo

> Ayudé a convertir Copilot de herramienta individual en práctica de equipo con reglas, ejemplos y revisión crítica.
