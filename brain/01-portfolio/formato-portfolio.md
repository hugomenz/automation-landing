# Formato del portfolio

## Comparativa

| Formato | Ventaja | Riesgo | Veredicto |
|---|---|---|---|
| Web personal simple | Control, CTA clara, se comparte fácil | Puede verse vacía si no hay casos | Usar |
| GitHub + README profesionales | Prueba técnica rápida | Vende mal a compradores no técnicos | Usar como respaldo |
| Notion público | Rápido | Menos comercial | Solo si necesitas salir hoy |
| PDF comercial | Fácil de adjuntar | No genera descubrimiento | Usar como apoyo |
| Landing page de servicios | Mejor para clientes | Necesita copy claro | Usar primero |
| Casos de estudio consultoría | Comunica seniority | Requiere síntesis | Obligatorio |

## Recomendación

Usaría primero una **landing page de servicios** con **casos de estudio estilo consultoría** y un **PDF comercial corto**.

## Estructura recomendada

### Home

- Hero con resultado.
- 3 dolores concretos.
- 3 servicios.
- 3 pruebas/casos.
- CTA.

### Servicios

- Internal tools.
- Angular/design systems.
- Automation/AI workflows.
- Copilot enablement.

### Casos de estudio

Cada caso debe tener:

- problema,
- contexto,
- intervención,
- stack,
- resultado,
- qué harías en un cliente real.

### Sobre mí

Breve. Enfocado en por qué puedes resolver esos problemas.

### CTA

Una sola acción:

> Agenda una llamada de 15 minutos.

### Página para cold outreach

URL simple tipo `/external-support`.

Contenido:

- para quién trabajas,
- dolores que resuelves,
- entregables de 1–2 semanas,
- CTA.

### Página para Upwork / plataformas

URL tipo `/freelance-profile`.

Contenido:

- skills principales,
- entregables claros,
- ejemplos,
- disponibilidad,
- tarifa orientativa o “available on request”.
